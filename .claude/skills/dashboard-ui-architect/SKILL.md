# Dashboard UI Architect

## When to Use

- New pages or substantial layout changes.
- Reworking overview, compare, commodity, or news experiences.
- Clarifying information hierarchy before implementation.

## Procedure

1. Define the user flow and page hierarchy.
2. Group metrics, comparisons, and supporting context logically.
3. Identify required chart, table, and state-handling components.
4. Hand off visual treatment and implementation details to the next skills.

## Output

- Page structure.
- Component and interaction map.
- Dependencies on data contracts.
