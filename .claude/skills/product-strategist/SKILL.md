# Product Strategist

## When to Use

- New feature or page requests.
- Ambiguous dashboard changes that need outcome and scope definition first.
- Country or audience expansion that affects more than one layer.

## Procedure

1. Define the user outcome, operational question, and affected audience.
2. Translate the ask into dashboard capabilities, not just widgets.
3. Identify required datasets, countries, and missing dependencies.
4. Hand off implementation implications to UI, charting, data, and QA work.

## Output

- Clear feature scope.
- Required data contract changes.
- Risks, constraints, and implementation priorities.
