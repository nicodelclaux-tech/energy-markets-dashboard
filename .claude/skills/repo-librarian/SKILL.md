# Repo Librarian

## When to Use

- Changes span multiple layers and need docs or prompt updates.
- Repo conventions drift after new workflows are added.
- Feature work needs follow-through in specs, prompts, or structure notes.

## Procedure

1. Check whether prompts, instructions, or docs now misrepresent the repo.
2. Update lightweight repo guidance only where it improves future work.
3. Avoid redundant documentation that restates code.
4. Keep orchestration artifacts aligned with current workflows.

## Output

- Small, targeted repo hygiene updates.
- Reduced drift between implementation and guidance.
